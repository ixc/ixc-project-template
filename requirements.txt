### Vendor ####################################################################

# If you need to fork and hack on any requirements so extensively that they
# will only be usable by this project, pull them into the ``vendor`` folder in
# this project with ``git subtree``. For example::
#
#    $ mkdir -p vendor
#    $ git subtree add -P vendor/ixc-blog git@github.com:ixc/ixc-blog.git master --squash
#
# Then install them directly from the vendor directory::
#
#    -e ./vendor/ixc-blog
#
# However, this should be avoided if at all possible, so that we can more easily
# contribute bug fixes and feature enhancements back upstream to these external
# projects.
